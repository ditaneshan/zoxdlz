# Elasticsearch 全文检索性能优化实战

## 引言
在高并发搜索场景中，Elasticsearch 的性能瓶颈往往不是“能不能搜到”，而是“能不能快且稳地返回结果”。全文检索涉及分词、倒排索引、相关性计算与分页排序，任何一个环节配置不当，都会放大 CPU、IO 和 JVM 压力。要系统优化，必须先理解底层原理，再针对查询、索引和集群层做协同调整。

## 核心原理分析
Elasticsearch 的全文检索依赖倒排索引：文本先被分析器切分为 term，再映射到文档列表。查询时，ES 通过 term 反查文档，并结合 TF/IDF、BM25 等算法计算相关性。性能优化的关键在于减少无效分词、降低大范围扫描、避免深分页和不必要的高亮计算。对于需要稳定延迟的业务，优先控制字段类型、分片数量、查询结构与缓存命中率。关于[最佳实践](https://home-ayx-app.com.cn)，通常应从“索引设计优先于查询优化”的思路入手。

## 代码示例
下面的例子展示了如何通过精简查询、限制返回字段、使用 `search_after` 替代深分页，来缓解全文检索延迟问题：

```json
POST /articles/_search
{
  "size": 20,
  "track_total_hits": false,
  "_source": ["id", "title", "summary"],
  "query": {
    "bool": {
      "must": [
        {
          "match": {
            "content": {
              "query": "分布式 搜索 优化",
              "operator": "and"
            }
          }
        }
      ],
      "filter": [
        { "term": { "status": "published" } }
      ]
    }
  },
  "sort": [
    { "_score": "desc" },
    { "publish_time": "desc" }
  ]
}
```

实践中还应配合：减少分片数、为低基数字段使用 `keyword`、避免通配符前缀查询、对热字段启用合适的索引模式，并通过慢查询日志定位耗时操作。

## 总结
全文检索优化不是单点技巧，而是索引、查询、分片与资源调度的系统工程。优先保证数据模型合理，再通过过滤条件前置、结果裁剪、分页策略优化和缓存利用降低成本，才能在复杂业务下保持稳定吞吐与低延迟。
